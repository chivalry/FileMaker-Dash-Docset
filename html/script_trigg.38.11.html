OnRecordRevert
Triggers a script to run before a set of records is reverted by the Records menu > Revert Record command or by the Revert Record/Request script step. This also triggers a script to run by reverting from a validation error dialog.
When the script runs
Before the event has been processed.
Modes in which the trigger can be used
Browse and Find modes.
Result
You can use the Exit Script script step within the triggered script to return True or False.
 •
True: The event that triggered the script proceeds.
 •
False: The event that triggered the script is canceled.
Originated in 
FileMaker Pro 10.0
Notes
 •
This trigger fires only when there are uncommitted record changes.
 •
Clicking Revert Field in a field validation error dialog box does not activate this script trigger.
 •
See Changing object data without activating script triggers for a list of other actions that do not activate this script trigger.
